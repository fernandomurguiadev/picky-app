# OpenSpec Design Skill

Habilidad para formalizar el diseño técnico de una solución dentro del flujo de cambios de OpenSpec.

## 🚀 Flujo de Ejecución
1.  Tomar el Master Plan aprobado por el Tech Agent.
2.  Generar el archivo `design.md` en la carpeta del cambio (`openspec/changes/KAN-XXX/`).
3.  Definir Entidades, Contratos de API y Componentes UI de forma detallada.
4.  Validar la paridad entre diseño y plan técnico.
