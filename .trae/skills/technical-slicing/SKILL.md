# Technical Slicing Skill

Habilidad para desglosar una funcionalidad compleja en tareas atómicas, manejables y testeables.

## 📋 Requisitos Previos
- Master Plan aprobado.
- Conocimiento de las capas del sistema (BE, FE, DB).

## 🚀 Flujo de Ejecución (Paso a Paso)

### Paso 1: Definición de Átomos
- Identificar la unidad mínima de código que puede ser probada independientemente (ej: un DTO, un componente standalone).

### Paso 2: Generación del Task List
- Crear o actualizar `tasks.md` en la carpeta del cambio.
- Usar nomenclatura `TSK-X.Y` (Fase.Tarea).
- Asegurar que cada tarea tenga un criterio de aceptación técnico.

## 🧪 Validación de Éxito
- Ninguna tarea toma más de 4 horas de implementación estimada (lógica).
- El progreso puede trackearse mediante checkboxes en Markdown.
